# Security Policy / 安全说明

## Reporting a security issue

Do not disclose a suspected credential, private address, personal path, or deployment weakness in a public issue. Use GitHub's **Report a vulnerability** form for this repository. Include only the minimum evidence needed to reproduce the problem.

请不要在公开 Issue 中粘贴疑似密钥、私有地址、本机路径或部署细节。请使用本仓库 GitHub 页面中的 **Report a vulnerability** 私密报告入口，并只提供复现问题所需的最少信息。

## Repository boundary

This public repository must never contain production credentials, origin addresses, SSH material, private backups, local workstation paths, databases, unpublished customer material, or private operational procedures. Contributors are responsible for reviewing both file content and image metadata before submission. GitHub secret scanning and maintainer review provide additional safeguards, but they do not replace that review.
