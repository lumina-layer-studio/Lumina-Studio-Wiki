# 打不开、一直转圈或生成失败，应该先查什么？

遇到问题时，先找出**最早出现异常的阶段**，再从那一步开始处理。不要同时换网络、改图片、清缓存、
改参数和重装切片器，否则即使突然恢复，也很难知道真正原因。

Lumina Studio 的完整流程可以分成六段：**打开页面 → 连接后端 → 准备最小输入 → 完成最小预览 →
生成文件 → 下载、打开或提交报告**。前一步没有成功，后一步通常也不会正常工作。

## 先不要做这四件事

- 不要在转圈时连续点击“预览”或“生成”；
- 不要把“清除缓存”当成第一步；
- 不要只改文件扩展名来伪装格式，例如把不支持的文件直接改名为 PNG；
- 不要把包含隐私、商业文件或完整日志的原始资料直接发到公开群聊。

先记录当前看到的状态、最早错误文字和操作步骤，再进行一次最小对照测试，通常比反复重试更快。

## 固定六步排查顺序

/>

### 第一步：页面能不能完整打开

如果浏览器直接报错、白屏，或很久都没有出现 Lumina Studio 界面，先不要检查图片和生成参数。
此时问题仍在入口层。

依次确认：

1. 使用的是当前官方入口，而不是旧收藏夹、群聊里的临时地址或本地开发地址；
2. 刷新后是否仍然完全打不开；
3. 同一时间换一个网络或设备，能否打开同一个入口；
4. 记录发生时间、所在地区和网络类型。

只有某个设备打不开时，优先检查该设备的浏览器、扩展、代理或网络；多个设备和网络同时打不开时，
更可能是服务端或线路问题。不要在没有对照结果时直接认定是图片、DNS 或账号问题。

### 第二步：看顶栏连接状态

页面刚打开时，短暂显示“正在检查后端…”属于正常过程。随后应显示“后端已连接”。

如果持续显示“后端不可达”：

- 在线版用户：记录时间、网络和页面状态，稍后重试或提交反馈；
- 本地版用户：通过正式启动入口重新启动 Lumina Studio，再重新打开页面；
- 不要在这一步反复更换图片、档案或生成参数，它们无法修复连接失败。

界面已经显示出来，不代表负责上传、预览和生成的服务一定可用。先让连接恢复，再继续后面的测试。

### 第三步：用最小输入区分“文件问题”和“系统问题”

准备一张尺寸较小、内容简单的 JPG 或 PNG，并使用你已经确认可用的耗材档案或 LUT。先关闭批量、
大尺寸分割和附加结构，只保留最基本的转换流程。

检查：

- 图片是否真的上传完成，而不是只显示了文件名；
- 来源类型是否选对，所需的档案、品牌或 LUT 是否已经选择；
- PNG 透明区、图片尺寸和文件内容是否正常；
- 如果是 SVG，是否存在字体、遮罩、滤镜、画布或复杂路径问题。

最小图片可以正常预览，而原文件不能，说明问题主要在原文件或当前参数组合。两者都不能预览，才继续
检查连接、来源和软件状态。SVG 的专项步骤见
[SVG 上传失败、白底、破洞和细节异常怎么排查？](pathname:///zh/docs/knowledge/svg-troubleshooting/)。

### 第四步：先让一次最小预览成功

首次排查时使用普通成品尺寸和默认参数，一次只启动一个预览任务，并等待本次任务明确完成或报错。

如果预览一直转圈或失败：

1. 确认顶栏仍显示“后端已连接”；
2. 确认图片和颜色来源都已就绪；
3. 暂时关闭大尺寸分割、批量、拼图和其他附加功能；
4. 换一张简单小图做一次对照；
5. 保留第一次出现的完整错误文字。

如果简单任务成功，再逐项恢复原图、尺寸和高级选项。每次只恢复一个条件，才能找到导致失败的那一项。

### 第五步：“生成”灰色或生成失败

Lumina Studio 需要先取得一次**有效的预览结果**，才能生成最终文件。因此“生成”按钮是灰色时，
通常不是按钮坏了，而是图片、来源或本次预览还没有准备好。

按下面顺序检查：

- 图片和颜色来源是否都处于就绪状态；
- 当前设置下是否已经成功完成一次预览；
- 修改图片、来源或关键参数后，是否重新预览；
- 当前模式、档案、附加功能和导出方式是否互相兼容；
- 成品尺寸、打印板范围或大尺寸分割设置是否触发了明确警告。

预览成功后仍生成失败，不要立即清缓存。先记录完整错误，关闭非必要的附加功能，再用同一张小图做
最小生成测试。

### 第六步：生成完成后，区分下载失败和切片器失败

生成成功后，先把 3MF 或 ZIP 完整下载到本机。然后再判断问题发生在哪里：

- 下载按钮失效或文件不存在：临时结果可能已经过期，重新预览并生成；
- 自动打开切片器失败：先确认文件已经下载，再检查设置中的切片软件和程序路径；
- 切片器能打开但材料不对：这是对象、项目耗材和真实料槽的映射问题；
- 大尺寸或批量任务：先完整解压 ZIP，再按说明处理其中的文件，不要只打开压缩包预览。

切片器映射的完整步骤见
[把 Lumina 3MF 正确导入切片器](pathname:///zh/docs/tutorials/3mf-slicer-material-mapping/)。如果文件能打开，
但预览和打印结果不同，请看
[为什么 Lumina 预览、切片器预览和打印实物不一样？](pathname:///zh/docs/knowledge/preview-slicer-print-differences/)。

如果问题发生在切片软件检测、macOS 手动应用路径、一键启动或 3MF 手动打开阶段，请看
[切片器没检测到，或者 3MF 打不开怎么办？](pathname:///zh/docs/knowledge/slicer-path-and-open-troubleshooting/)。

## 什么时候才应该清除缓存

“清除缓存”只适合本地版中的临时会话、预览或下载状态已经异常，而且重启和最小测试仍无法恢复的情况。
它不是网络诊断工具，也不能修复错误图片或不兼容参数。

使用前需要知道：

- 清除后，当前临时预览和下载链接可能失效；
- 需要重新上传、预览并生成；
- 它不会替你修正档案、LUT、切片器或真实耗材映射；
- 正常的用户耗材档案和配置库不应因为清理临时缓存而被删除。

如果你不确定问题是否来自缓存，先完成本文的最小对照测试，并保留错误资料。

## 常见错误文字代表什么

不需要记住错误编号，先看它表达的是哪一类问题：

| 错误含义 | 常见原因 | 优先处理 |
| --- | --- | --- |
| 输入无效或缺少必填信息 | 没有选来源、参数组合不完整、文件内容异常 | 回到最小输入，逐项补齐 |
| 不支持这种文件 | 文件内容与格式不符，或当前流程不支持 | 使用真实导出的 JPG、PNG、SVG、WebP 或 HEIC |
| 文件过大 | 上传体积或处理复杂度超出限制 | 缩小副本做对照，不要覆盖原图 |
| 文件或会话不存在 | 临时结果过期、页面刷新或会话已失效 | 重新上传、预览并生成 |
| 当前任务过多 | 连续重复点击，或服务正在处理其他任务 | 停止点击，等待后再执行一次 |
| 服务暂时不可用或超时 | 后端、线路或任务处理异常 | 记录时间和最早错误，再重试或反馈 |

不要只发“报错了”或错误编号。完整错误文字、发生阶段和复现步骤比编号更有用。

## 一次最小复现测试应该怎么做

建议固定使用下面这组条件：

1. 当前官方入口或本地正式启动入口；
2. 一张简单、小尺寸、确认可正常打开的 JPG 或 PNG；
3. 一个已经使用成功的耗材档案或 LUT；
4. 普通成品尺寸与默认参数；
5. 关闭批量、大尺寸分割和附加功能；
6. 只执行一次预览，成功后只执行一次生成。

这组条件成功后，再逐个换回原图、来源、尺寸和高级选项。最先让问题重新出现的条件，就是最值得继续
检查的范围。

## 提交反馈前保留这些资料

如果仍然无法解决，请准备：

- Lumina Studio 版本、在线版或本地版；
- 发生问题的准确时间、地区和网络类型；
- 问题停在哪一步；
- 第一条完整错误文字；
- 从打开页面开始的准确操作步骤；
- 原始输入文件，或一个不含隐私但能够复现的最小文件；
- 当前来源类型、档案/LUT、模式、尺寸和关键选项；
- 必要的页面截图；
- 如果已经生成，保留原始 3MF 或 ZIP。

软件内提交错误报告时，可以先展开“自动收集的信息”进行检查。报告会包含用于定位问题的页面状态、
当前选择、最近操作和必要日志；截图是可选的。不要主动附上密码、密钥、私人聊天记录或与问题无关的
商业文件。如果在线提交连续失败，可以复制备用报告后再通过官方渠道发送。

## 常见问题 Q&A

### “正在检查后端…”显示多久算异常？

刚打开页面时短暂出现是正常的。如果很久都不变，或随后明确显示“后端不可达”，就先按连接问题处理，
不要继续改图片和参数。

### “生成”按钮一直是灰色，是不是权限问题？

通常不是。先确认图片、颜色来源和必要选项都已就绪，并且当前设置下完成过一次成功预览。修改关键设置
后，旧预览可能不再有效，需要重新预览。

### 为什么一个网络能打开，另一个网络打不开？

这说明图片本身通常不是首要原因。记录两个网络、发生时间和所在地区，再提交给维护者判断入口、线路或
服务状态。不要在没有证据时同时修改 DNS、代理、浏览器和系统设置。

### 清缓存会删除我的耗材档案吗？

正常的“清除缓存”针对临时会话、预览和输出，不应删除用户耗材档案与配置库。但当前临时预览和下载
可能失效，所以清理后需要重新生成。

### 为什么刷新页面后，刚才的下载链接失效了？

预览和生成结果可能属于临时会话。页面刷新、会话切换、超时或清缓存后，旧链接可能不再有效。重新上传
或选择输入，完成预览并再次生成即可。

### 提交错误时一定要上传我的原图吗？

不一定。优先提供能复现问题的最小文件。如果原图含有隐私、商业素材或未公开作品，可以制作一个不含
敏感内容、但保留相同格式和问题特征的示例，并说明原图与示例的差异。
