# 从位图生成 SVG，再发送到图像转换

如果手里只有一张 PNG、JPG、WebP 或 BMP，但图案本身是边界清楚的 Logo、图标、线稿或
纯色色块，可以先用 Lumina Studio 的“图像转矢量”工具，把像素重新描摹成 SVG 路径，
再继续进入图像转换。

这一步适合解决“我只有位图，但确实需要路径”的问题。它不会把任何图片无损变清晰，
也不会自动生成已经匹配好耗材的打印文件。不了解两类文件差别时，先阅读
[位图与矢量图有什么区别？](pathname:///zh/docs/knowledge/bitmap-vs-vector/)。

:::info 本篇做什么
本篇只讲完整主流程：上传位图、设置矢量化参数、检查原图与 SVG、下载 SVG，或把结果
直接发送到“图像转换”。照片、复杂绘画和连续渐变通常更适合保留位图。
:::

## 开始前先判断素材是否合适

### 通常适合尝试矢量化

- Logo、图标、文字轮廓和简单标志；
- 边界清楚、颜色数量较少的纯色色块图；
- 背景干净、线条明确的线稿；
- 需要在矢量编辑器中继续改路径、节点或填充色的图案。

### 通常不应强行矢量化

- 照片、毛发、皮肤、云雾和复杂材质纹理；
- 大量柔和阴影、半透明、发光和连续渐变；
- JPG 压缩痕迹或噪点很重的低清图片；
- 把像素格当作造型语言的像素画。

自动矢量化是在**根据现有像素重新猜轮廓**。低清原图里已经不存在的眼睛、文字、尖角
或纹理，不会因为输出文件改成 SVG 就自动回来。开始前保留未经修改的原始位图，不要
覆盖唯一原件。

## 第一步：上传位图

1. 打开 Lumina Studio 顶部的“图像转矢量”；
2. 把 PNG、JPG、WebP 或 BMP 拖入上传区，也可以点击上传；
3. 确认上传区已经显示正确图片；
4. 第一次先保持默认参数，点击“开始矢量化”。

<img
  src={require('./assets/01-upload-and-defaults.webp').default}
  alt="在图像转矢量页面上传位图并保留自动颜色、默认平滑度和默认细节设置"
/>

先跑一次默认结果，通常比一开始同时改很多参数更容易判断问题来自哪里。默认状态下，
颜色数量由软件自动判断，平滑度为中间值，统一细节控制处于禁用状态。

## 第二步：理解三个基础参数

| 参数 | 它影响什么 | 什么时候改 |
| --- | --- | --- |
| 颜色数量 | 决定允许保留或归并成多少类颜色 | 自动结果出现太多近似色、杂色路径，或你明确只需要固定几种颜色时，切换到手动 |
| 平滑度 | 控制轮廓抽稀与平滑程度 | 边缘过于毛躁时可逐步提高；小字、尖角和细线开始消失时应退回 |
| 细节等级 | 启用后用一个滑块统一联动多项细节参数 | 不想逐项调整高级参数，但需要快速比较“更简洁”和“保留更多细节”的结果时使用 |

颜色数量不是“打印时必须准备的耗材卷数”。它描述的是这一次 SVG 里识别到的颜色类别；
后续进入图像转换后，仍要根据所选 LUT、耗材和建模方式重新预览。

参数没有适合所有图片的固定答案。Logo 可能只需要几种明确颜色，复杂插画可能包含许多
色阶。以重要轮廓和最终预览为准，不要只追求颜色数越多或形状数越多。

## 第三步：并排检查原图与 SVG

矢量化完成后，页面会同时显示“原始图片”和“SVG 预览”。不要只确认任务成功，要逐项
检查这两张图之间发生了什么变化。

<img
  src={require('./assets/02-compare-result.webp').default}
  alt="Lumina Studio 并排显示原始位图和自动矢量化后的 SVG 结果"
/>

建议按下面顺序观察：

1. **外轮廓**：斜边、圆角和尖角有没有明显变形；
2. **小字与细线**：字腔、笔画和窄线是否仍然连贯；
3. **封闭区域**：眼睛、孔洞和独立色块有没有被填平或断开；
4. **背景**：透明背景、压缩噪点或纸张纹理有没有变成多余形状；
5. **渐变**：连续颜色会被拆成有限色带，分区是否还能接受；
6. **整体观感**：在实际成品尺寸下，结果是否比直接使用位图更符合目标。

本篇演示图在默认参数下得到 **155 个形状、24 种颜色、900 × 600 像素画布**。这些数字
只用于说明页面怎样显示结果，不是推荐值，也不代表每个形状都会成为一个独立打印件。

:::warning
如果重要文字、脸部或细线已经在 SVG 预览中丢失，后续生成 3MF 不会自动把它们恢复。
应先返回这里调整参数，或改用原始位图和高保真模式。
:::

## 结果不理想时，按顺序调整

每次只改一个参数，再重新矢量化并与上一版比较。这样才能知道哪项调整真正有效。

| 看到的问题 | 优先尝试 | 不要急着做 |
| --- | --- | --- |
| 大量近似颜色和小碎片 | 手动减少颜色数量，或小幅提高平滑度 | 一次把所有高级过滤值拉到最大 |
| 细线、小字或尖角消失 | 降低平滑度，撤回过强过滤，再比较细节控制 | 只因为 SVG 文件更小就接受明显缺失 |
| 色块之间有细小白缝 | 保持“输出增强”中的描边与覆盖修复，必要时小幅调整描边宽度 | 过度加粗描边，让相邻颜色互相覆盖太多 |
| 边缘仍然毛躁 | 小幅提高平滑度，再检查曲线和尖角 | 把低清锯齿误认为能被无损恢复的细节 |
| 形状数很高、处理变慢 | 减少颜色、清理源图噪点，或适度过滤小区域 | 只看文件体积判断复杂度 |
| 渐变出现明显色带 | 这是有限色区的正常结果；比较原位图的高保真预览 | 继续增加大量路径，试图还原无限连续渐变 |

“输出增强”主要用于减少色块接缝并保留部分细线；“高级参数”包含预处理、超像素分割、
曲线拟合和区域过滤。普通素材先保持默认值。只有确认基础参数不能解决具体问题时，再
展开对应分组，一次只调整一项并保留可回退的版本。

## 第四步：下载 SVG，或直接继续图像转换

结果区域会显示形状数、颜色数、画布像素和颜色样本，并提供两个不同用途的按钮。

<img
  src={require('./assets/03-result-actions.webp').default}
  alt="矢量化结果区域显示形状数、颜色数，并提供发送到图像转换和下载 SVG 两个按钮"
/>

### 下载 SVG

适合这些情况：

- 想把文件留档，之后再回到 Lumina；
- 需要在 Inkscape、Illustrator 或其他矢量编辑器中清理节点；
- 要修改某些路径、描边、填充或画布范围；
- 想把矢量化结果与原始位图作为两个独立版本保存。

下载得到的是 `vectorized.svg`。建议立刻改成能辨认内容和版本的文件名，例如
`logo-vector-v1.svg`，不要让多次测试结果全部使用相同名称。

### 发送到图像转换

确认 SVG 预览可以接受后，点击“发送到图像转换”。软件会：

1. 读取刚生成的 SVG；
2. 把它作为当前素材载入图像转换；
3. 自动切换到“矢量模式”；
4. 保留后续选择来源、LUT、尺寸、结构和生成方式的入口。

<img
  src={require('./assets/04-vector-mode.webp').default}
  alt="发送完成后图像转换页面自动选择矢量模式"
/>

这一步只是完成了**素材交接**，不是已经生成 3MF。进入图像转换后还要：

1. 选择传统 LUT、反推 LUT 或耗材档案等来源；
2. 选择实际使用的 LUT 或耗材组合；
3. 设置成品宽高、厚度和单双面结构；
4. 先点击“预览”，检查颜色、色块和层叠；
5. 预览正确后，再生成并进入切片器验证。

不同来源怎么选，可参考
[现成耗材档案、自制档案、反推 LUT 与传统 LUT 怎么选？](pathname:///zh/docs/knowledge/material-profile-selection/)；
模式选择可参考
[高保真、像素艺术与矢量模式怎么选？](pathname:///zh/docs/knowledge/image-mode-selection/)。

## 为什么 SVG 颜色不等于实际耗材？

矢量化阶段识别的是源图中的屏幕颜色和色块边界。打印阶段还会受到实际耗材、层高、
底色、层叠顺序、透光特性、喷嘴线宽和切片路径影响。

因此：

- SVG 中有 24 种颜色，不代表必须准备 24 卷耗材；
- SVG 看起来颜色正确，不代表切片器或实物一定完全相同；
- 自动矢量化只负责整理图形结构，不负责替你完成耗材校准；
- 最终判断仍要经过 Lumina 预览、切片器逐层预览和必要的小尺寸试打。

预览之间的区别见
[为什么 Lumina 预览、切片器预览和打印实物不一样？](pathname:///zh/docs/knowledge/preview-slicer-print-differences/)。

## 常见问题 Q&A

### 转成 SVG 后是不是一定更清晰？

不是。SVG 可以按新尺寸重新绘制路径，但路径来自对现有像素的重新判断。原图中的锯齿、
压缩块和错误边缘也可能被描成路径。只有轮廓判断正确时，缩放优势才有意义。

### 颜色数量应该设成多少？

没有统一答案。第一次使用自动模式，再根据预览判断。图标和 Logo 可尝试固定为接近原始
设计的颜色数；复杂图像如果需要大量颜色才能勉强接近原图，通常应同时比较高保真位图
工作流。

### 形状数越多越好吗？

不是。更多形状可能保留了细节，也可能只是把噪点、压缩痕迹和渐变拆成大量碎片。要看
重要结构是否正确，以及文件能否顺利编辑、预览和生成。

### 为什么渐变变成一条一条的色带？

矢量化需要用有限数量的填充区域近似连续变化。增加颜色可以让过渡更细，但也会增加路径
和复杂度。若连续渐变是作品重点，保留原始位图并比较高保真模式通常更合适。

### 发送到图像转换后，为什么仍提示选择 LUT？

因为 SVG 只描述图形路径和显示颜色，并不知道你手上有哪些耗材，也没有完成打印配方。
选择 LUT 或耗材档案后，Lumina 才能把图像颜色映射到实际层叠方案。

### 发送后还能回到原始位图吗？

可以，但应依靠你事先保存的原始文件。矢量化结果不是原图的无损备份，不能保证反向恢复
全部像素、渐变和纹理。

### SVG 上传后白底、破洞或细节异常怎么办？

先确认矢量化预览是否已经异常。如果这里正常、进入图像转换后才出现问题，再按
[SVG 上传失败、白底、破洞和细节异常怎么排查？](pathname:///zh/docs/knowledge/svg-troubleshooting/)
检查画布、背景、路径、蒙版、滤镜和文件复杂度。
