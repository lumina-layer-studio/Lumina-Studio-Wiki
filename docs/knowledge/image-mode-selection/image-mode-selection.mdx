# 高保真、像素艺术与矢量模式怎么选？

如果只想先看答案：**先判断原图最重要的特征，再选择负责保留这种特征的模式。**

- 照片、复杂插画、柔和渐变和纹理，通常先看 **高保真**；
- 像素画、方格图案和马赛克风格，通常先看 **像素艺术**；
- 原生 SVG 的 Logo、文字、图标、线稿和纯色色块，通常先看 **矢量模式**。

这里的“通常”不是强制规则。三种模式保留的东西不同，不是从低到高的质量排名。
同一份素材可以分别预览，再按照成品尺寸、画面风格、模型复杂度和个人喜好选择。

## 三种模式分别保留什么？

  src={require('./assets/mode-comparison.webp').default}

/>

上图是原理示意，用来解释三种建模方式关注的内容，并不是把同一文件导入 Lumina 后的
最终输出对比。真实结果还会受到颜色数量、成品尺寸、喷嘴线宽、切片参数和原图质量影响。

### 高保真：保留连续颜色、纹理与细节变化

高保真模式会把输入画面当作连续图像处理，重点保留颜色之间的变化、纹理、光影和渐变。
它常见于这些素材：

- 人物、宠物、风景和产品照片；
- 有笔刷纹理、阴影、发光或柔和渐变的复杂插画；
- 透明背景的复杂 PNG；
- 虽然来自 SVG，但希望按照最终画面而不是原生路径处理的内容。

“高保真”不代表所有细节都能原样打印。原图中比喷嘴线宽更细的睫毛、小字和噪点，仍然
可能在模型生成或切片时合并。颜色变化非常多时，模型区域和计算量也可能增加，因此应该
结合最终尺寸检查预览，而不是只看原图在屏幕上是否清楚。

### 像素艺术：保留方格结构与有限色板

像素艺术模式把规则像素格当作画面结构的一部分，适合这些素材：

- 像素画角色、道具和场景；
- 方格拼图、马赛克、十字绣式图案；
- 经过有意降色、每个色块边界都落在网格上的图片；
- 希望成品保留清楚方格感的设计。

普通照片使用像素艺术模式后，连续的脸部、天空和阴影会被拆成明显方块。这不一定是错误，
也可以是一种风格选择；但它不会自动让照片更清楚。

准备像素画时，尽量保留原始网格，不要先用平滑缩放把边缘变模糊。如果需要放大源图，使用
最近邻缩放更容易保留每个方格的边界。导入后还要检查一个源像素在目标尺寸下是否足够大，
避免切片后因线宽限制而消失。

### 矢量模式：按 SVG 路径重建边界

矢量模式使用 SVG 中的路径、轮廓和填充信息生成几何，适合这些素材：

- 原生 SVG Logo、图标和标识；
- 已经转换为路径的文字；
- 边界清楚的线稿；
- 颜色较少、轮廓明确的纯色色块图形。

当前 Lumina Studio 中，矢量模式只对 SVG 输入开放。它的优势是边界来自路径，不需要先把
轮廓压成固定像素数量；改变成品尺寸时，也更容易保持曲线和斜边平滑。

不过，文件名以 `.svg` 结尾并不代表它一定适合矢量模式。SVG 可以嵌入位图，也可以包含
滤镜、蒙版、复杂渐变、剪切路径、描边效果和其他高级特性。不同解析方式对这些特性的支持
并不完全相同，导入后要重点检查缺失区域、填充颜色、孔洞、重叠关系和文字外观。

## 常见素材怎么选？

| 素材情况 | 可以先查看的模式 | 重点检查 |
| --- | --- | --- |
| 人物、宠物或风景照片 | 高保真 | 五官、毛发、阴影、背景噪点和小色块 |
| 有纹理和柔和渐变的复杂插画 | 高保真 | 渐变是否断层、细碎纹理是否过多 |
| 原生像素画 | 像素艺术 | 方格是否对齐、缩放后有没有被平滑 |
| 马赛克或方格拼图 | 像素艺术 | 每格尺寸能否满足喷嘴线宽 |
| 原生 SVG Logo 或图标 | 矢量模式 | 路径、孔洞、重叠、填充和画布范围 |
| 文字与线稿 | 矢量模式或高保真 | 是否已转路径、线条和小字能否打印 |
| 透明背景 PNG | 按画面内容判断 | 白边、灰边、半透明像素和背景杂点 |
| 自动描摹得到的 SVG | 矢量模式与高保真都可比较 | 杂乱节点、色块数量和错误轮廓 |

这张表给的是检查起点，不是最终结论。比如一个只有三种纯色的 PNG Logo，可以直接使用
高保真；如果同时拥有设计软件导出的原生 SVG，也可以用矢量模式比较边界和模型结构。
两种预览都满足需要时，按照后续编辑习惯和个人观感选择即可。

## 透明 PNG 应该选哪一种？

透明 PNG 仍然是位图，透明背景不会自动把它变成矢量图。

- PNG 里是照片、复杂插画或柔和渐变时，可以先查看高保真；
- PNG 里是原生像素画时，可以先查看像素艺术；
- PNG 里是 Logo 或线稿，但你没有原生 SVG 时，可以先用高保真，也可以另存一份自动
  描摹的 SVG 与矢量模式结果比较。

透明边缘经常带有白边、灰边或半透明脏像素。在深色预览中不明显的杂边，生成模型后可能
变成独立色块。导入前最好把图片放在深色和浅色背景上各检查一次。

## 先清理原图，再判断模式

  src={require('./assets/source-cleanup.webp').default}

/>

### 透明边缘

去掉不需要的白边、灰边和半透明像素。需要保留柔和边缘时，不要为了追求“干净”把所有
半透明区域都直接删掉，而是先查看高保真预览是否符合画面需要。

### 杂点与小碎片

删除不属于主体的孤立色块、压缩噪点和扫描灰尘。它们可能在生成后形成许多很小的独立区域，
增加模型复杂度，却不会给成品带来有效细节。

### 过细线条与小字

按计划打印的实际尺寸检查线条和文字。屏幕放大后能看见，不代表缩到 60 mm 或 100 mm 后
仍能被喷嘴线宽保留。需要时可以适当加粗线条、放大文字或简化过小的装饰。

### 渐变、滤镜与蒙版

渐变、阴影、模糊、发光和复杂蒙版依赖连续画面时，可以先保留位图效果查看高保真。若一定
要使用 SVG 路径，导入前应检查这些效果是否已经正确展开，或者准备一份视觉效果已经固定的
位图版本用于比较。

:::note
清理原图不会增加原本不存在的细节。它的作用是减少杂边、碎片和在目标尺寸下必然消失的
小元素，让模型生成更接近你真正想保留的画面。
:::

## 四个容易混淆的地方

### 转成 SVG，不等于恢复细节

自动矢量化会根据现有像素重新猜测轮廓。它可能让简单 Logo 的边缘更规整，也可能把照片的
噪点、压缩方块和锯齿变成大量路径。源图中已经丢失的眼睛、文字和纹理不会自动回来。

### 原图像素越高，不等于成品一定越清楚

高分辨率原图能提供更多信息，但最终细节仍受成品尺寸、喷嘴线宽、颜色区域和切片路径限制。
如果想理解尺寸与线宽的关系，可以继续阅读
[0.2 和 0.4 喷嘴相当于多少分辨率？](pathname:///zh/docs/knowledge/nozzle-equivalent-resolution/)。

### 矢量模式不等于颜色更多

矢量路径擅长描述边界，不会自动增加可用耗材颜色，也不会把纯色色块变成照片级渐变。最终
能表达多少颜色，仍然取决于当前材料、颜色配置和生成结果。

### 像素艺术不等于低质量

像素画主动使用有限网格和色板，它的方格边缘就是设计语言。用平滑算法把它变成模糊曲线，
反而可能破坏原本的造型。更多基础区别可以参考
[位图与矢量图有什么区别？](pathname:///zh/docs/knowledge/bitmap-vs-vector/)。

## 一分钟判断顺序

拿到素材后，可以按下面顺序检查：

1. **先看内容**：它是照片、复杂插画、像素画，还是 Logo、文字和线稿？
2. **再看内部结构**：SVG 中是否真的有可用路径，还是只嵌入了一张位图？
3. **确认想保留什么**：连续颜色、规则方格，还是平滑路径边界？
4. **清理输入**：检查透明杂边、噪点、小碎片、过细线条和小字。
5. **按实际尺寸预览**：重点看五官、文字、细线、孔洞和独立小色块。
6. **必要时比较两个模式**：不要根据模式名称判断质量，以预览和实际试打为准。

## 常见问题 Q&A

### 单独上传 SVG 后，为什么软件自动选择了矢量模式？

当前版本会优先把单独上传的 SVG 交给矢量模式，方便保留路径信息。这只是默认起点，不代表
这个 SVG 一定适合路径建模。若文件包含复杂渐变、滤镜或嵌入位图，也可以切换到高保真比较
画面效果。

### JPG、PNG 和 WebP 能不能使用矢量模式？

不能直接作为矢量路径使用。这些格式保存的是像素。需要矢量模式时，要先取得原生 SVG，
或者用描摹工具生成候选 SVG，再检查路径是否准确。单纯把扩展名改成 `.svg` 没有作用。

### Logo 一定要转成 SVG 吗？

不一定。清晰、尺寸足够的 PNG Logo 可以使用高保真。原生 SVG 的优势是更容易保留路径边界
和修改尺寸；如果自动描摹后的 SVG 反而产生杂边或错误色块，继续使用原 PNG 也完全合理。

### 像素画能不能使用高保真？

可以，但要比较方格边界是否仍符合预期。像素艺术模式更明确地把网格当作结构；高保真则会
把它当作一般图像处理。最终选择取决于你想保留方格感，还是希望获得另一种平滑或区域效果。

### 矢量模式导入后少了一部分图形怎么办？

先检查 SVG 是否包含嵌入位图、滤镜、蒙版、剪切路径、描边效果、未转路径文字或超出画布的
内容。可以在矢量编辑器中把重要外观展开为普通路径和填充，再另存为新文件导入；也可以导出
一份高分辨率 PNG，用高保真比较最终画面。

### 三种模式都能生成时，应该以什么为准？

以目标尺寸下的预览、模型复杂度、重要细节、切片结果和个人观感为准。三种模式没有统一的
优先级，也不需要为了追求某个名称而固定选择其中一种。
