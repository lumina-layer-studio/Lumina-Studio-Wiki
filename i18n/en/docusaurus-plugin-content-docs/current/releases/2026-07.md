# Lumina Studio 2.0 Development Report: July 2026

> Ongoing · Current through July 17, 2026

[← June 2026](/en/docs/changelog/2026-06) · [Changelog overview](/en/docs/changelog)

## Monthly overview

July continues to improve cloud resources, pixel/material previews, and vector-model export, while introducing native SVG relief. The month is not sealed yet: only merged, evidence-checked work appears here, and features that remain at the design stage are explicitly labelled as in development.

## Highlights

### Native SVG relief

SVG artwork can now enter the relief workflow directly, retaining vector boundaries while generating different model heights. The relief backing was also made watertight, reducing non-manifold or open-backing warnings in slicers.

<!-- devlog-entry:2026-07-native-svg-relief prs:510,511 -->

### Cloud LUT installation and review polish

Installing a cloud LUT now refreshes its details, list entry, and status. Multi-source selection, missing resources, and official/community badges are handled consistently, while feedback records each selected source for reproduction.

<!-- devlog-entry:2026-07-cloud-lut-polish prs:497,500 -->

## New features

- Cross-platform browser notifications can alert users when a task completes, so long previews and generations do not require watching the active tab continuously.
  <!-- devlog-entry:2026-07-browser-completion-notifications prs:504 -->
- Images can be rotated, with incompatible combinations restricted in vector mode to keep preview and final coordinates aligned.
  <!-- devlog-entry:2026-07-image-rotation prs:450 -->

## Color Workstation

- Changing a material gradient or target material clears previous replacement choices, preventing stale colours from appearing in confirmation.
  <!-- devlog-entry:2026-07-material-replacement-state prs:493 -->

## Calibration and materials

- Wujing materials were renamed to Karvax. Existing materials, recipes, and backing selections migrate automatically without requiring users to select them again.
  <!-- devlog-entry:2026-07-karvax-rename prs:507,508 -->
- Duplicate-material deletion now explains whether the current duplicate or a shared resource is affected, reducing accidental removal.
  <!-- devlog-entry:2026-07-duplicate-material-deletion prs:502 -->

## Vectorisation and modelling

- Material-profile vector previews now use the real layer count, and 5-colour vector exports retain the correct base layers, reducing preview/export layer mismatches.
  <!-- devlog-entry:2026-07-material-vector-layers prs:499,506 -->
- Preview-cache estimates for large SVG artwork better match real cost, avoiding premature rejection of processable files.
  <!-- devlog-entry:2026-07-vector-cache-estimates prs:496 -->

## Preview and interaction

- Pixel mode preserves source canvas size and pixel palettes, producing more accurate backing dimensions and fewer speckles in material previews.
  <!-- devlog-entry:2026-07-pixel-preview-fidelity prs:488,495 -->

## Slicers and 3MF

- Snapmaker U1 project information for OrcaSlicer is more complete, with reliable defaults when some machine fields are unavailable.
  <!-- devlog-entry:2026-07-snapmaker-u1-metadata prs:494 -->

## In development

### Cloud Resource Center V2

Requirements and architecture have been designed for a new resource centre that would unify browsing, installation, and updates for materials, LUTs, and future resources. Only design documents have merged so far, so this is not listed as a delivered feature.

<!-- devlog-entry:2026-07-cloud-resource-center-design prs:503,509 -->

## Stability, performance, and infrastructure

Cloud connection guidance and in-app announcements were also consolidated so resource counts, connection behaviour, brand migration, and user notices have one maintenance path.

<!-- devlog-entry:2026-07-engineering-foundation prs:501 -->

## Known limitations

- This page remains ongoing and will be sealed only after the end-of-month review.
- Cloud Resource Center V2 remains a design; the existing cloud material and LUT library entries stay in use.
- Native SVG relief still needs broader verification across complex paths, holes, and material-layer combinations.
