# Large Format: Tile Size, Assembly Order, and ZIP Files

:::note[Translation notice]
This page is an **AI-assisted translation** of the Chinese article. The terminology,
links, calculations, and screenshots were checked against the current English interface.
:::

Large Format divides one finished artwork into adjacent tiles, generates a separate 3MF
for every tile, and packages the results in a ZIP. It is intended for wall art, signs,
display panels, and other work larger than one usable build plate.

Remember these rules:

- **Width and Height** describe the assembled artwork; **Tile Width and Tile Height**
  are maximum dimensions for one generated part;
- filenames run from top to bottom and left to right: `R1C1` is the upper-left tile;
- the last row or column is often smaller than the maximum tile size;
- tiles are adjacent and non-overlapping. Lumina does not add joints, locating holes,
  overlap, or assembly clearance;
- Preview checks the complete artwork, not physical seams or real bed placement;
- after download, open every 3MF in the slicer and verify size, direction, materials,
  and toolpaths before printing.

The real example below uses a finished size of `450×300 mm` and a tile limit of
`180×180 mm`. Lumina generated a `3×2` set of six 3MF files.

## When should I use Large Format?

Use it when:

- the complete artwork exceeds the printer’s usable bed area;
- every tile should share one colour source, structure, and colour-layer height;
- you can align and secure the printed pieces manually;
- you are prepared to slice, label, and inspect every tile separately.

Do not use it when each part needs different materials or structure, or when you expect
automatic puzzle joints or alignment hardware. First prove the image, colours, and basic
structure with an ordinary single-image project.

## Step 1: enter the assembled size

Upload the image, choose the colour source, and enter the final Width or Height. With
aspect ratio locked, the other value follows the source image.

This is the size **after assembly**, not the size of one tile. If the composition or
orientation is wrong, prepare the source first; do not use tiling to stretch the artwork.
See [Upload, Replace, Crop, and Rotate an Image](pathname:///en/docs/knowledge/image-upload-crop-rotate/).

## Step 2: enable Large Format and set tile limits

<img
  src={require('./assets/01-large-format-settings.webp').default}
  alt="Real English Lumina Studio interface with finished size, Large Format Mode, tile limits, and a 3 by 2 calculation highlighted"
/>

Enable **Large Format Mode**, then set **Tile Width** and **Tile Height**. These are
maximums, not guaranteed dimensions for every part.

For this example:

- finished size: `450×300 mm`;
- maximum tile: `180×180 mm`;
- columns: `ceil(450 ÷ 180) = 3`;
- rows: `ceil(300 ÷ 180) = 2`;
- total: `3 × 2 = 6` tiles.

The interface shows `3×2 = 6 (90×120mm)`. The value in brackets is the remaining width
of the last column and remaining height of the last row. It does not mean all six tiles
are 90×120 mm.

| Tile position | Actual size |
| --- | --- |
| R1C1, R1C2 | 180×180 mm |
| R1C3 | 90×180 mm |
| R2C1, R2C2 | 180×120 mm |
| R2C3 | 90×120 mm |

Set tile limits from the printer’s **usable** bed area, not only its advertised size.
Leave room for excluded zones, Brim, purge or wipe areas, clips, and slicer-added
geometry. Confirm a safe size in the real slicer first.

## Step 3: read the coordinate map

<img
  src={require('./assets/02-tile-map.webp').default}
  alt="Concept diagram showing a 450 by 300 millimetre artwork split into R1C1 through R2C3"
/>

- `R` means row and increases from top to bottom;
- `C` means column and increases from left to right;
- `R1C1` is the upper-left corner;
- `R2C3` is the lower-right corner in this example.

Keep the generated filenames. After printing, label the back of each part or its storage
bag with the coordinate. Renaming similar pieces without a record makes orientation and
assembly unnecessarily difficult.

:::warning[No automatic seam design]
Large Format cuts along adjacent boundaries. It does not create overlap, registration
pins, puzzle tabs, adhesive gaps, or shrinkage compensation. Add and test those features
in the source design or an external modelling tool when required.
:::

## Step 4: Preview the complete artwork

<img
  src={require('./assets/03-preview-whole-image.webp').default}
  alt="Real English Lumina Studio complete artwork Preview with layer thumbnails highlighted"
/>

Run **Preview** and check:

1. orientation, aspect ratio, and visible boundary;
2. colour source and intended filaments;
3. missing colours, broken edges, or incorrect transparency;
4. Layer Preview order and colour-layer count;
5. outline, backing, and other enabled structure.

This Preview describes the whole artwork. It does not show six separate build plates,
physical seams, or whether a tile fits the selected printer. Fix colour, edge, or layer
problems before generating the ZIP.

For the roles of Lumina Preview, slicer Preview, and the physical print, see
[Why Lumina Preview, Slicer Preview, and the Print Differ](pathname:///en/docs/knowledge/preview-slicer-print-differences/).

## Step 5: Generate once and save the ZIP

Select **Generate** once and wait for all tiles. Do not repeatedly select the button,
refresh, or replace the image while generation is running.

<img
  src={require('./assets/04-download-zip.webp').default}
  alt="Real English Lumina Studio result with six generated Large Format tiles and Download ZIP highlighted"
/>

The verified example returned six files in a ZIP of about 16.9 MB:

```text
large-format-demo_R1C1.3mf
large-format-demo_R1C2.3mf
large-format-demo_R1C3.3mf
large-format-demo_R2C1.3mf
large-format-demo_R2C2.3mf
large-format-demo_R2C3.3mf
```

Immediately verify that:

- file count matches the tile count;
- coordinates cover the complete grid;
- no file is empty or unreadable;
- the ZIP and extracted files are saved outside the temporary browser result.

Generated download links are not permanent storage. Save the ZIP before refreshing,
starting a new task, or waiting for the temporary result to expire.

## Step 6: inspect every tile in the slicer

Open each 3MF as a complete project and check:

- millimetre dimensions against the tile table;
- placement inside the usable bed area;
- display face, backing, and rotation;
- printer, nozzle, layer height, and real filament mapping;
- continuous colour toolpaths and no missing thin walls;
- Brim, outline, or support that might alter a seam edge.

Use one printer profile, nozzle, layer height, filament batch, and orientation wherever
possible. Bed texture, material state, shrinkage, and manual alignment can still create
visible seams even when digital boundaries are adjacent.

For material mapping, see
[Import a Lumina 3MF into the Slicer Correctly](pathname:///en/docs/tutorials/3mf-slicer-material-mapping/).

## Step 7: dry-fit before permanent assembly

Arrange the first row from `R1C1` to the right, then continue with the next row. Confirm
orientation and image continuity before using a backboard, fixture, tape, adhesive, or
mechanical fastener. Start from one reference corner and progress row by row so alignment
error does not accumulate in several directions.

The correct fixing method depends on material, backing thickness, environment, load, and
service life. Test adhesion and safety for the real installation.

## Common questions

### Why is the last tile only 90×120 mm?

Because 180×180 mm is a maximum. After two 180 mm columns, 90 mm of the 450 mm width
remains; after one 180 mm row, 120 mm of the 300 mm height remains.

### Can all tiles have exactly the same dimensions?

Yes, if both finished dimensions divide evenly by the chosen tile limits. You may also
add deliberate margin in the source design, but do not distort the artwork merely to
make the numbers tidy.

### Does Lumina add overlap or locating features?

No. Design overlap, cutting allowance, locating holes, or joints before Large Format and
include their footprint in bed-fit calculations.

### Preview looks correct. Why does one tile exceed the bed?

The full-image Preview is not a real slicer layout. Reduce the tile limit or reposition
the object after considering excluded zones, Brim, and printer profile rules.

### One 3MF will not open. Must I redo every tile?

Keep the downloaded ZIP and all readable files, then generate the same task again. If the
same coordinate fails repeatedly, inspect that region for tiny geometry or unsupported SVG
features. See [Slicer Detection and 3MF Open Troubleshooting](pathname:///en/docs/knowledge/slicer-path-and-open-troubleshooting/).
